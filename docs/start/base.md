# 基础介绍

## 项目背景

因为信息差的关系，当前的各个大学校内有各种各样的墙层出不穷。

对大多数这些墙而言，都是基于微信号、QQ号、公众号等来制作的，很少本身有自己的技术能力，多数是靠着运营团队分批次管理自己的账号来做的。

本身这些平台采用的多是用户发消息给qq号或者微信号，然后运营人员收集用户的诉求，统一发送到朋友圈或者qq空间。

为了解决这些团队在运营过程中的高人力成本（需要经常查看消息并且定期发表），以及用户侧在发布消息后需要等待回复的体验，我们开发了该平台。

在保证用户体验的同时，也减少了运营人员的压力，并且由于完全开放小程序测源码，各团队完全可以自主决定是否使用我们提供的默认样式。
